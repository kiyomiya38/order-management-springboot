# Day5（7/15）コードリーディング & テスト & アカウント管理

## 目的（Day5でできるようになること）
- コードを読んで全体像を説明できる
- テストを実行できる
- 管理者でアカウント管理ができる

---

## 1. フォルダ構成を読む
読むべき場所:
- `domain` : Entity
- `repository` : DBアクセス
- `service` : 業務ルール
- `web` : 画面遷移
- `templates` : 画面

---

## 2. テストを実行
```powershell
mvn test
```

確認:
- `AttendanceServiceTest` が通る
- 出勤成功 / 二重出勤失敗の2本

---

## 3. アカウント管理（管理者）
1. admin / admin123 でログイン
2. `http://localhost:8080/users` を開く
3. 新規ユーザー作成 → 一覧に反映
4. 編集・削除を確認

---

## 4. 管理者による勤怠編集
1. `http://localhost:8080/admin/attendances` を開く
2. 任意の勤怠を「編集」
3. 状態と時刻を変更して保存

---

## 5. 受講者向けまとめ
「このシステムは何をしているか」を説明できる状態にする:
- 出勤/退勤の状態管理
- 業務ルール違反は画面で通知
- DBに1日1件の勤怠

---

## 6. チェックポイント
- テストが通る
- adminでアカウント管理ができる
- 一般ユーザーで勤怠操作できる
- adminで勤怠編集ができる

---

## 7. 時間割目安
- 午前: コードリーディング(90分) / テスト(30分)
- 午後: アカウント管理(45分) / 勤怠編集(45分) / まとめ(30分)
