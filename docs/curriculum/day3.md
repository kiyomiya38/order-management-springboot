# Day3（7/13）退勤と業務ルール（Day3コード使用）

## 目的（Day3でできるようになること）
- 退勤を実装し、状態遷移が分かる
- 業務ルール違反時のエラー表示を理解できる

## 使用するコード
`stages/day3` を使う（退勤・ルール実装）

---

## 1. 起動
```bash
cd stages/day3
mvn spring-boot:run
```

## 2. 出勤 → 退勤
1. 出勤  
2. 退勤  
3. 状態が「退勤済み」になる

## 3. ルール違反の確認
- 未出勤で退勤 → エラー
- 退勤済みで再退勤 → エラー

---

## 4. コード確認ポイント
- `AttendanceService#clockOut`
- `BusinessException`
- `index.html` のエラー表示

---

## 5. 時間割目安
- 午前: 退勤実装理解(60分)
- 午後: ルール違反の再現(60分)
