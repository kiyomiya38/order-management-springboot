# 講師向けガイド

## 進め方のポイント
- Java文法の深追いは避け、Spring Bootの流れと設定変更を重視
- 後続のDocker/CI/CD/K8sで使い回すことを前提に、設定キーとログ出力を意識
- Day5は「コードを読む」ことが主目的なので、正解をすぐ見せずに追わせる

## よくある詰まり
- Java 17のPATH/JAVA_HOME設定
- 8080ポート競合
- `application.yml` のインデント
- ログイン情報の入力ミス

## 補足説明の例
- `application.yml` の ${VAR:default} は環境変数で上書きできる
- profile切替は `SPRING_PROFILES_ACTIVE` を使う
- H2は後でRDS等へ差し替え可能な設計にしている

## 進捗が早い受講者向け追加課題
- 勤怠のCSV出力を追加
- 管理者の勤怠一括修正画面を追加
- 退勤時刻のバリデーション強化（開始時刻より前を禁止）

## 評価観点・採点基準（例）

### 必須達成
- Spring Bootが起動し、勤怠トップが表示できる
- 出勤/退勤が動作する
- 勤怠一覧が表示できる
- ルール違反で画面にエラーが出る
- 管理者でアカウント管理ができる（Day5）

### 加点項目
- 勤怠ルールを言語化できる
- H2コンソールでデータを確認できる
- 設定の差分（dev/prod）を理解している

### 参考配点（20点満点）
- 起動/画面表示: 4点
- 出勤/退勤: 6点
- 勤怠一覧: 4点
- ルール理解/説明: 4点
- 設定理解: 2点
